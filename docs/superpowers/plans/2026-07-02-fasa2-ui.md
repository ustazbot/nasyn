# Fasa 2 (Part 1): Real Visual UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace Fasa 1's plain/unstyled UI with the real PRD-designed
dark-mode UI (Boot, Home, Prayer Session, Summary) matching the
`NASYN-app-sample-design/*.png` wireframes, without touching any
`lib/prayer/`, `lib/audio/`, or `lib/guided/` logic from Fasa 1.

**Architecture:** A `theme/` package (colors + text styles) and an
`i18n/` package (BM/EN string map behind a Riverpod `StateProvider`) are
shared foundations. Each screen is rebuilt on top of Fasa 1's existing
`GuidedModeController`/`PrayerStateEngine`/`AudioCueResolver` — this plan
only touches `lib/ui/`, adds `lib/theme/`, `lib/i18n/`, and one new
`lib/prayer/prayer_recitation_text.dart` (a pure data map, not FSM logic).

**Tech Stack:** Flutter/Dart, flutter_riverpod (already a dependency from
Fasa 1) — no new packages.

## Global Constraints

- Package/project name: `nasyn_app`
- Do NOT modify `lib/prayer/prayer_state.dart`, `lib/prayer/prayer_config.dart`, `lib/prayer/prayer_state_engine.dart`, anything in `lib/audio/`, or `lib/guided/guided_mode_controller.dart` — all Fasa 1 logic, unchanged
- Palette (PRD §11): `primaryTeal` `#0D4F4F`, `accentGreen` `#52B788`, `accentGold` `#C9A84C`, `accentBlue` `#3B82C4` (sampled from wireframe, exact value not in PRD text), `darkBg` `#0F1A14`, `errorRed` `#C0392B`, `lightText` `#F7F9F7`
- Text sizing (PRD §11): Display 56–72px bold, Body 32–40px medium, Label 24px minimum
- Dark mode only for this phase (light mode deferred)
- No new dependencies — BM/EN via a plain `Map`, not `flutter_localizations`/`intl`
- Assistance spectrum is a tap-based 3-zone segmented control, NOT a draggable slider (Bos's decision — safer for elderly users)
- Vision Mode toggle is visually present but disabled ("Coming Soon") — Vision Mode doesn't exist until Fasa 3
- Prayer Session screen gets no widget test (real `AudioPlayerService`/`audioplayers` platform-channel dependency, same constraint Fasa 1 hit) — verified by `flutter analyze`/`flutter build apk --debug` + manual device check instead

import 'dart:async';

import 'package:flutter/material.dart';

import '../i18n/app_locale.dart';
import '../i18n/app_strings.dart';
import '../theme/app_colors.dart';
import '../theme/app_text_styles.dart';
import 'home_screen.dart';

class BootScreen extends StatefulWidget {
  const BootScreen({super.key});

  @override
  State<BootScreen> createState() => _BootScreenState();
}

class _BootScreenState extends State<BootScreen> {
  Timer? _timer;

  @override
  void initState() {
    super.initState();
    _timer = Timer(const Duration(seconds: 4), () {
      if (!mounted) return;
      Navigator.of(context).pushReplacement(
        MaterialPageRoute(builder: (_) => const HomeScreen()),
      );
    });
  }

  @override
  void dispose() {
    _timer?.cancel();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      backgroundColor: AppColors.darkBg,
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            Image.asset('assets/images/logo-nasyn.png', width: 160),
            const SizedBox(height: 16),
            Text(
              AppStrings.of('tagline', AppLocale.bm),
              style: AppTextStyles.body,
            ),
          ],
        ),
      ),
    );
  }
}
